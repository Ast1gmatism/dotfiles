## Installer

### install.sh (MVP)
- [ ] Точка входа для запуска через `curl ... | bash` 🟢
- [ ] Проверка ОС (Arch и деривативы через /etc/os-release), иначе явный exit с ошибкой 🟢
- [ ] Bootstrap минимальных зависимостей (git и т.п., если не установлены) 🟢
- [ ] Клонирование репозитория в фиксированный путь (`~/.local/share/<project>`, имя уточнить) 🟢
- [ ] Установка пакетов: hyprland, quickshell, sddm, kitty, nautilus, firefox 🟢
- [ ] Базовые GPU-пакеты без vendor-детекта (mesa) 🟢
- [ ] Сбор списка пакетов, которые не удалось установить — без прерывания процесса 🟢
- [ ] Backup существующих конфигов перед заменой (простое переименование в `.bak.<timestamp>`, без merge) 🟢
- [ ] Copy-деплой `configs/hypr` → `~/.config/hypr` 🟢
- [ ] Copy-деплой `configs/quickshell` → `~/.config/quickshell` 🟢
- [ ] Деплой SDDM-темы (`configs/sddm/custom-theme` → `/usr/share/sddm/themes/...`) 🟢
- [ ] Деплой `configs/sddm/sddm.conf` 🟢
- [ ] Enable `sddm.service` 🟢
- [ ] Enable базовых сервисов (NetworkManager, bluetooth, pipewire/wireplumber) 🟢
- [ ] Финальное сообщение: что установлено / что зафейлилось / что делать дальше 🟢

### update.sh (отдельный скрипт, не часть install.sh)
*Должен ощущаться нативной частью системы, а не "зайди и сделай git pull" — по образцу imperative-dots.*

- [ ] Commit-aware update: сравнение локального и upstream коммита ⚪
- [ ] Partial update по `git diff` между коммитами (copy только изменённых файлов) ⚪
- [ ] Full reinstall fallback: если tracked commit отсутствует/битый — полная перезапись ⚪
- [ ] Обработка удалённых upstream файлов (диф с diff-filter=D, удаление устаревших) ⚪
- [ ] Логика apply-once для записей из апстрима (см. свой PR) ⚪
- [ ] Сохранение локальных пользовательских правок при обновлении managed-файлов ⚪
- [ ] Backup перед каждым обновлением (timestamped) ⚪
- [ ] Восстановление user-owned файлов после обновления (colors.json и т.п.) ⚪
- [ ] Проверка новых пакетов-зависимостей при обновлении (upstream добавил пакет → доставить) ⚪
- [ ] Запуск update из QuickShell (виджет/индикатор «доступно обновление» — пересекается с идеей pacman-виджета) ⚪
- [ ] Ветка dev/master как флаг (--dev) для тестирования перед релизом ⚪
